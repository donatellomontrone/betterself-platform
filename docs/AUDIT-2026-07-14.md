# BetterSelf audit - 2026-07-14

## Verified

- TypeScript, ESLint, local production build, and Vercel Preview build pass.
- Public routes return successfully; private dashboard, messages, and admin
  routes redirect signed-out visitors to Clerk.
- Direct treatment booking keeps the selected treatment and starts at patient
  details instead of showing the catalog again.
- Neon contains 26 active treatments and has no orphan bookings, orphan
  payments, missing checkout references, duplicate transaction references, or
  paid bookings without a paid payment row.
- PayMongo webhook and reconciliation routes validate signatures, references,
  ownership, and cancelled-booking state.
- Calendly reconciliation uses a database-backed lock and can be triggered by an
  admin with visible results and an audit entry.
- Public treatment recommendations are rate-limited in Neon and have a local
  fallback when no AI provider is configured.
- Mobile and desktop public pages have no horizontal overflow or broken images.
- Cookie consent remains explicit while using a compact mobile layout that does
  not cover the primary hero content.
- Global responses send content-type, referrer, frame, permissions, and minimal
  content-security headers.
- `npm audit --omit=dev` reports no known dependency vulnerabilities.

  treatment guidance. The safe catalog-based fallback remains active otherwise.
